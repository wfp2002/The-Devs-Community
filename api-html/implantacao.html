Implantação


Agora que você implementou as principais integrações com a API da Ambev, é hora de realizar a implantação. Esse processo consiste em validar as integrações implementadas (homologação) e, obtenção de um token com acesso à produção.

Homologação com a Ambev



Para que sua aplicação possa utilizar as APIs de produção, ela deverá passar por um processo de homologação, onde as integrações mais relevantes serão verificadas e avaliadas. Para solicitar a homologação de sua APP quando já estiver com sua implementação concluída, execute todo o fluxo de operação no seu ambiente de sandbox, conforme listado abaixo, e em seguida abra um chamado no portal dos desenvolvedores solicitando a homologação e informando o nome do (Owner) do access_token que você utilizou para realizar seus testes e também as URLs dos serviços de notificação de pedidos, consulta de frete e reserva de estoque, se foram implementados. Nossa equipe irá revisar todos os requests enviados e fazer os testes nas URLs para então aprovar a app para operar em ambiente de produção.

Os requests a serem verificados serão os seguintes:

POST /loads/products
GET /loads/products
GET /sellerItems
PUT /sellerItems/{skuSellerId}/status
PUT /sellerItems/{skuSellerId}/prices
PUT /sellerItems/{skuSellerId}/stock
POST /orders
GET /orders/status/new
PUT /orders/status/approved/{orderId}
GET /orders/status/approved
POST /orders/{orderId}/trackings/sent
POST /orders/{orderId}/trackings/delivered

Para maiores detalhes acesse: Fluxos Básicos de Integração Processo de homologação

Obtenção do Token de Produção


Até agora, todas as integrações foram realizadas com um token gerado automaticamente no ambiente de sandbox. Para o ambiente de produção esse token não será gerado automaticamente. Em vez disso, esse token poderá ser solicitado via um processo OAuth2.

Fluxo OAuth 2

OAuth 2 é um padrão de mercado e garante que informações mais sensíveis como usuários e senha não sejam diretamente utilizados por aplicativos de terceiro. Visão Geral sobre o oAuth na API do Marketplace Fluxo de comunicação OAuth2
